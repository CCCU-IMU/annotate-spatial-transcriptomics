# Standard sheep-ovary R-first annotation workflow

Use this contract whenever the resolved context is sheep/Ovis/ovine plus ovary/ovarian and a full-feature Seurat RDS is available. It converts the successful R/Scanpy strategy into a low-freedom execution path. Parameters outside this contract require a recorded user-approved exception.

## Fixed preprocessing and resolution contract

Treat a StereoPy-converted `cellbin_PPed` RDS as a raw-count container. Rebuild `Spatial counts -> SCT v2/glmGamPoi -> PCA50 -> 30-PC SNN(k=30, Annoy/cosine) -> UMAP`, and never reuse imported StereoPy PCA/UMAP.

Use the fixed `0.2,0.4,0.6,0.8` BANKSY grid for fresh verified same-batch whole-tissue preprocessing. When the selected input is an existing BANKSY result, review the complete grid actually stored in that hash-bound input. Use the separate `0.1,0.2,0.3,0.4,0.6` grid for every query-derived broad-class or targeted reclustering cohort; do not substitute either grid for the other. Bind the distinction in `config/annotation_contract.json` and validate the BANKSY choice with `validate_banksy_broad_resolution_selection.py --annotation-contract ...`. Do not recluster final QC. Values below 0.1 remain forbidden for query-derived reclustering; repair a defective graph rather than lowering resolution.

Run `scripts/validate_resolution_grid.py` before every whole-tissue, broad-class or targeted-cohort submission. Every completed cohort records the exact candidate grid and selected resolution. A missing grid or an out-of-contract value invalidates the result.

## Ordered biological phases

1. **Whole-tissue broad pass.** Generate complete DEG, marker/anti-marker, UMAP, whole-section spatial and per-cluster highlights for the normal grid. When BANKSY memberships are used, review every resolution for all catalog lineages, zero-census default candidates and observation-level embedded programs in large clusters. Freeze the integrated-evidence optimum for initial broad annotation: preserve broad-lineage recall and large-label purity first, avoid state/technical fragmentation second, and use lower complexity only as a tie-breaker. Validate the BANKSY decision artifact before writing initial labels.
2. **Clusterwise open-world initial broad annotation.** At the selected resolution, validate that every default lineage has at least two explicit positive families, then run `init_open_world_lineage_audit.py` and review every catalog boundary plus every unexplained multi-gene program. Use the machine-derived lineage decision table to assign supported coarse resident parents; this pass does not have to separate every final ovarian lineage. An informative mixed cluster returns to its numerically dominant supported parent, commonly Stromal/mesenchymal when appropriate, while coherent subthreshold epithelial, vascular, mature-smooth or other programs remain `watch`. Absolute full-feature detection/prevalence and pseudobulk establish a shared parent program; centered module scores and one-vs-rest DEG are comparative evidence and cannot alone reject an abundant tissue-wide lineage. Low-information, featureless or irreducibly mixed clusters enter `qc_holdout`. Do not create intermediate biological containers at this stage.
   Repeat the catalog-by-cluster screen at the next two higher available resolutions. Record weak coherent programs as `watch`; failure to pass the broad naming threshold does not establish absence.
3. **One reclustering cohort per initial broad class.** Submit an adaptive query-only reclustering job for every initial broad class and run the complete grid. Use `broad_purity_audit`, where one stable compartment is valid only after exact-membership purity passes. If no stable hidden lineage or true subtype exists, close successfully as `homogeneous_parent_confirmed`; cohort identity alone cannot return all observations to the parent. A class too small for defensible reclustering closes as `not_applicable_reviewed`, remains broad-only and records a hash-bound limitation artifact.
   For the selected cohort resolution and next two higher available candidates, score every subcluster against all sheep-ovary candidate lineages plus unexplained programs. Do not limit a Stromal cohort to stromal subtypes, an Immune cohort to immune subtypes, or any cohort to its parent branch.
4. **Subcluster and observation-subset adjudication.** Rescore every selected-resolution subcluster against the complete catalog. Whole-subcluster return is a high-purity fast path with an absolute support floor and minimum margin. When a follicular-wall or other subcluster is mixed, freeze independently supported Smooth muscle, Stromal/mesenchymal, Vascular-associated, Theca or other disjoint observation subsets using the full `observation-subset-writeback.md` contract; remove them and rescore the exact remainder. The aggregate winner does not veto a subset, and ACTA2/TAGLN or location alone cannot define Smooth muscle. After broad lock, run a default complete fine pass for every present parent: audit all machine-actionable optional children at the selected parent-cohort resolution plus adjacent resolutions, not only vascular children. Shared parent markers cannot decide children; require a child discriminator program, anti-program clearance and compatible topology. State/intensity differences remain tags. Do not manufacture subtypes to match a reference.
5. **Direct cross-lineage return.** If a subcluster from one broad-class run coherently supports another lineage—for example a pregranulosa-like subcluster found during Oocyte-candidate reclustering—write it directly to the supported target broad/fine label. Preserve source run, source cluster, cohort, membership hash and evidence. Do not create an intermediate cohort and do not automatically recluster it again with the target class.
6. **Decision-relevant mixtures only.** When sufficient observations contain interpretable competing biological programs, create a temporary `targeted_recluster_cohort`; Oocyte contamination-safe review is one such cohort. If targeted reclustering remains insufficient, calibrated RCTD is lower-priority assistance: canonical high plus independent evidence may support a fine call, moderate may support broad-only, and low returns to QC holdout. A featureless or uninterpretable mixture goes directly to QC holdout.
7. **All-cell broad Atlas concordance, without QC reclustering.** After every broad-class and targeted decision is terminal, freeze residual `qc_holdout` and map the complete analysis set once using the declared external reference. Without a matched count-level reference, GSE233801 is primary. An unlabeled frozen-QC observation with calibrated moderate-or-higher, non-OOD, ontology-compatible, in-scope prediction returns broad-only with `fine_anchor_eligible=false`; marker/spatial evidence remains an audit or coherent-group challenge, not a duplicate per-cell gate. Lower/OOD predictions remain QC. Already-defined cells are compared, not overwritten; material broad disagreement or coherent OOD triggers one complete-cluster/cohort query-evidence review.
8. **Post-Atlas broad-class completeness review.** For every present broad class, check complete accepted membership marker/anti-marker, DEG, spatial morphology and observation-level purity. For every zero-census default sheep-ovary lineage, inspect all-cell marker localization, selected-plus-two-higher scans, embedded large-label components, QC/OOD, spatial morphology and technical missingness. Atlas is challenger-only and cannot establish absence. Residual QC at least 10% or 50,000 observations reopens the upstream initial broad-recall audit before completion.
9. **Final single annotation.** Every analysis-set observation receives either one moderate-or-higher `final_broad_label` with an optional high-confidence `final_fine_label`, or an explicit retained QC/technical state. Apply only catalog-bound, high-confidence fine assignments with `build_reliable_fine_annotation.py`, then require `validate_final_fine_annotation.py` to pass. A fine writeback cannot alter its locked parent broad label. Every nonempty fine label must use `defined_fine` for both active and final state; broad-only parents and observations retain empty fine labels without synthetic placeholders. Preserve assignment mode, source lineage, confidence, state/spatial/QC tags and evidence. Do not create strict/inclusive/display copies.
10. **Completion, main-Agent quality approval and lightweight review.** Require exact project-local expression ancestry, analysis-set ownership, terminal broad-class/targeted cohorts, validated cross-lineage returns, a terminal residual-QC Atlas phase, current open-world and broad-completeness audits, zero open incidents, current profile/preset binding, normal-grid compliance, final taxonomy audit and a fresh direct-workflow completion gate. Generate only the frozen broad spatial and canonical-marker evidence assets. The main Agent then reviews biological quality. Only after a pass build the lightweight user-confirmation HTML; final DEG, full tree dotplots, per-node/per-gene maps and release HTML wait for user confirmation.
   Completion additionally requires `validate_lineage_signal_coverage.py` PASS. The lightweight review contains canonical marker-expression spatial panels across all analysis-set cellbins, independent of assigned labels, so a missing or overbroad lineage remains visually auditable.

## Final evidence and report cohort

All observations formally returned to a biological broad class—including direct cross-lineage returns and calibrated Atlas broad-only rescue—participate in that class's final DEG, canonical marker dotplot, data-specific marker dotplot, UMAP and spatial maps. Only observations with a real high-confidence fine label participate in fine-label DEG/dotplots. Retained QC, technical and initial-QC states remain outside biological DEG.

Publish one annotation tree and one final census. Follow the successful Scanpy report layout: overall annotated spatial map above an expandable tree; per-node jump buttons to spatial highlights; broad and fine marker panels; marker-gene spatial maps grouped by supported cell type; route/threshold/outcome tables; a top link to the workflow; and the detailed Chinese workflow/state reconstruction at the bottom.

## Failure and resource contract

Before submission, parse every generated R, Python, shell and AIP file with `scripts/preflight_generated_job.py`. Record every EXIT, cancellation, fail-closed gate, syntax error, dependency problem and resource mismatch before accepting a repair. A sample cannot complete with an open incident.

Reuse validated immutable SCT/BANKSY reductions and per-resolution memberships by hash. Do not reread or recompute large expression objects for a postprocessing-only repair. Match requested CPUs to real workers and do not assume every BANKSY/Leiden stage uses all allocated cores. Delay release figures and HTML until the user confirms the frozen first annotation.
