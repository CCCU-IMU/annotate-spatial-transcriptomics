# Standard sheep-ovary R-first annotation workflow

Use this contract whenever the resolved context is sheep/Ovis/ovine plus ovary/ovarian and a full-feature Seurat RDS is available. It converts the successful R/Scanpy strategy into a low-freedom execution path. Parameters outside this contract require a recorded user-approved exception.

## Fixed preprocessing and resolution contract

Treat a StereoPy-converted `cellbin_PPed` RDS as a raw-count container. Rebuild `Spatial counts -> SCT v2/glmGamPoi -> PCA50 -> 30-PC SNN(k=30, Annoy/cosine) -> UMAP`, and never reuse imported StereoPy PCA/UMAP.

Use `0.1,0.2,0.3,0.4,0.6` as the complete formal candidate grid for both whole-tissue clustering and every biological/QC pool reclustering. Select adaptively within that grid. Do not run, rank, interpret or write back `0.01`, `0.02`, `0.05` or another below-grid resolution. If cluster inflation occurs, audit the graph, input reduction, neighbor construction, connected components and imported metadata; do not lower the resolution to conceal a graph defect.

Run `scripts/validate_resolution_grid.py` before every whole-tissue or pool submission. Every terminal anchor-recluster route records the exact candidate grid and selected resolution. A missing grid or an out-of-contract value invalidates the route.

## Ordered biological phases

1. **Whole-tissue broad pass.** Generate complete DEG, marker/anti-marker, UMAP, whole-section spatial and per-cluster highlights for the normal grid. Freeze the lowest-complexity solution that preserves supported broad lineages.
2. **Direct high-confidence anchors.** Freeze only coherent full-feature marker programs with compatible spatial morphology. Do not treat the remaining graph clusters as labels.
3. **Open-world lineage discovery.** Run `init_open_world_lineage_audit.py` after the initial decision ledger exists, then review every boundary in `sheep_ovary_candidate_lineage_catalog.json`: follicular/germline, steroidogenic, stromal–mesenchymal–contractile/mural, vascular/lymphatic, immune, epithelial/mesothelial and neural/glial/neuroendocrine. Also audit every unexplained multi-gene program not represented in the catalog. This is a boundary checklist, not a label quota: `not_supported` is valid, but omission is not. A missing class in the deidentified reference case is never negative evidence. Supported candidates create sample-specific biological pools and enter ordinary anchor/evidence routes; unsupported candidates receive a negative audit. Rarity alone is neither a cell type nor a route trigger. Validate the current ledger- and catalog-bound audit with `validate_open_world_lineage_audit.py`.
4. **Route A — large/continuous unresolved populations.** Return them to a canonical competing-axis parent pool, add balanced reference-only anchors, and run query-only graph/UMAP/DEG on the normal grid. Each query observation becomes a broad return, a high-confidence fine call, an interface/QC/context-specific route or a versioned successor pool.
5. **Route B — local lineage interfaces.** Run targeted anchor reclustering first. Record RCTD/reference applicability. Extreme RCTD plus independent evidence may support a high-confidence fine call; high confidence may support broad-only. Medium/low and other unresolved interface observations enter the frozen post-clustering QC holdout; they do not call Atlas directly. A reviewed `not_applicable` result does not skip targeted current-query anchors.
6. **Route C — residual low-information/QC holdout only.** Freeze the complete non-zero holdout assembled after biological/interface routes, run one full QC-pool balanced-anchor reclustering on the normal grid, and return every supported cluster first. Freeze cells still retained in QC as a distinct residual child snapshot. Only that exact residual membership may then use GSE233801 as the primary public somatic Atlas plus internal anchors, marker/anti-marker and observed-density spatial evidence. The Atlas route must bind the prior QC-anchor attempt ID, outcome hash, parent/child snapshots and identical residual membership hash. Moderate-or-higher calibrated consensus may return a broad class. Lower confidence remains QC/review. Atlas is not run routinely for defined broad classes, fine labels or ordinary biological pools. Record an explicit zero-holdout audit when Route C has no query.
7. **Oocyte-specific contamination-safe route.** Only when the open-world audit finds an Oocyte candidate, send the complete multi-module starting pool to query-only reclustering. Strict seeds and compact foci are support, not membership. Release Oocyte only from coherent non-ZP/maternal/ooplasm enrichment, somatic anti-program separation and compatible object morphology. Cortical/edge location is not negative evidence. Reroute somatic-dominant clusters with ambient/adjacent tags. Other context-specific lineages use ordinary Route A/targeted validation rather than a generic rare-cell route.
8. **Final single annotation.** Assign one `final_broad_label` only at moderate-or-higher confidence and one optional `final_fine_label` only at high confidence. Broad-only rescue has `fine_anchor_eligible=false`. Preserve route, confidence, source, state/spatial/QC tags and assignment evidence as metadata; do not create strict/inclusive/display label copies.
9. **Completion, main-Agent quality approval and lightweight review.** Require exact analysis-set ownership, all applicable routes closed, explicit phase audit including reviewed not-applicable/zero-query phases, a current open-world lineage audit, zero open incidents, current biological/profile-preset binding, normal-grid compliance, final taxonomy audit and a fresh completion gate. Generate only the frozen broad spatial and canonical-marker evidence assets. The main conversation Agent then performs a concise biological quality comparison with the validated deidentified R-first process reference; this occurs after all annotation and is never a broad-pass approval. A result may pass with documented concerns or return to targeted iteration. Only after a pass build the lightweight user-confirmation HTML. Generate final DEG, full tree dotplots, per-node/per-gene spatial assets and release HTML only after user confirmation.

## Final evidence and report cohort

All observations formally returned to a biological broad class, including calibrated broad-only rescue, participate in that class's final DEG, canonical marker dotplot, data-specific marker dotplot, UMAP and spatial maps. Only observations with a real high-confidence fine label participate in fine-label DEG/dotplots. Retained interface, QC, technical and initial-QC states remain outside biological DEG.

Publish one annotation tree and one final census. Follow the successful Scanpy report layout: overall annotated spatial map above an expandable tree; per-node jump buttons to spatial highlights; broad and fine marker panels; marker-gene spatial maps grouped by supported cell type; route/threshold/outcome tables; a top link to the workflow; and the detailed Chinese workflow/state reconstruction at the bottom.

## Failure and resource contract

Before submission, parse every generated R, Python, shell and AIP file with `scripts/preflight_generated_job.py`. Record every EXIT, cancellation, fail-closed gate, syntax error, dependency problem and resource mismatch before accepting a repair. A sample cannot complete with an open incident.

Reuse validated immutable SCT/PCA/SNN and per-resolution memberships by hash. Do not reread or recompute large expression objects for a postprocessing-only repair. Match requested CPUs to real workers; one Leiden optimization and unwrapped `FindAllMarkers` are single-threaded. Delay release figures and HTML until the user confirms the frozen first annotation.
